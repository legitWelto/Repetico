# Repetico – Agent Design Guidelines

## 🎨 Adaptive UI & Styling

The interface intelligently morphs to fit your device and environment.

### Responsive Layouts

| Orientation / Size | Layout Strategy |
|---|---|
| **Portrait (Mobile)** | Streamlined, stack-based single-column view. Primary interaction is the main player card. |
| **Landscape (Mobile)** | Wide-screen layout that optimizes the timeline and section buttons into a side-by-side arrangement. |
| **Desktop Grid** | Single unified screen displaying all 3 panels simultaneously. Left: Main player (without intervals bar). Top Right: Song Settings. Bottom Right: Interval Settings (with Master Interval toggle in its header). |

### Theme

- **Dark & Light Modes:** Full theme support that respects the system `prefers-color-scheme` media query, with an optional manual toggle override.

## 📐 Layout & Positioning Rules for Agents

### Cards Always Use Relative Positioning

> **Rule:** All card-level components MUST use **relative units** for sizing and positioning.

- Use `%`, `vw`, `vh`, `svh`, `em`, `rem`, `clamp()`, `min()`, `max()` — never hard-coded `px` values for width, height, padding, or font sizes on card or layout elements.
- Low-level detail elements (e.g. border-radius, icon stroke-width, shadow blur) may use `px` where truly fixed pixel density is needed.
- **Circular dials and turntables** (the play-button ring and speed-ring) must be sized in `vmin` or `%` of their parent, so they scale correctly in both portrait and landscape.
- Section buttons must fill available horizontal space via `flex: 1` or `grid` with `auto-fill` / `auto-fit` columns — never a fixed pixel width.

### UI Uses All Available Space

> **Rule:** The UI should always consume 100% of the available viewport.

- The root layout container must be `width: 100%; height: 100svh;` (or `100dvh` with `100vh` fallback).
- Cards use `flex-grow: 1` / `grid` tracks with `1fr` so empty space is never wasted.
- In portrait mode the main player card expands to fill the vertical space between the header and section-button bar. The sections grid fills any remaining space.
